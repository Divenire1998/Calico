Calico is a lightweight visual-\/inertial calibration library that allows for rapid problem construction, debugging, and tool creation. Unlike other codebases that are limited to standalone calibration tools for specific hardware setups, Calico is a flexible library that enables {\bfseries{building and modifying custom tools}} to suit your hardware requirements. If your calibration problem aligns with our geometry paradigm, there is no need to modify the underlying optimization.

Calico features\+:


\begin{DoxyItemize}
\item Flexible auto-\/differentiable cost functors.
\item Sensor intrinsics, extrinsics, and latency estimation.
\item The ability to add multiple sensors.
\item Measurement outlier tagging and exclusion.
\item Per-\/sensor robustifier kernels.
\item Ability to create custom sensor models.
\end{DoxyItemize}

Check out our \href{https://github.com/yangjames/Calico/wiki}{\texttt{ wiki pages}} for more info.\hypertarget{md__home_james_Documents_calico-work_Calico_README_autotoc_md1}{}\doxysection{License}\label{md__home_james_Documents_calico-work_Calico_README_autotoc_md1}
Poor sensor calibration is a problem endemic to robotics, yet it\textquotesingle{}s typically not given enough attention because it tends to detract from higher level project goals. The purpose of this library is to help roboticists quickly resolve their calibration issues so that they can work on more interesting things.
